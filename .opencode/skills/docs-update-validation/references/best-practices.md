# Validation 更新のベストプラクティス

## 何を書くか

- いつ検証するか
- 何で検証するか
- 期待する結果
- 問題が起きたときの扱い

## 変更時の必須追従

- テストコードをすべて変更する
- spec と実装コードへの影響を確認する
- 必要なら ADR を更新する
- 必要なら技術スタックを更新する

## 良い validation の条件

- 検証タイミングが明確
- 実行方法が再現可能
- 成功条件が観測可能
- 失敗時の扱いが曖昧でない

## 更新時の作法

- まず共有候補を提示して、どれを採用し何で検証するか確認する
- architecture と矛盾しないことを確認する
- 実際のテスト手段が存在するか確認する
- 変更した validation とテストコードを同時に更新する
