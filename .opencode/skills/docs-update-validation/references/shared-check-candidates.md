# 共有して確認する検証候補

## 候補項目

### 仕様整合性

- requirement / spec / architecture と実装内容が矛盾していないか

### 回帰防止

- 既存機能や既存フローを壊していないか

### エラー時の安全性

- 失敗時に壊れ方が安全か

### データ整合性

- 入出力、状態遷移、保存データが壊れないか

### 境界条件

- 空値、最小値、最大値、重複、順序差で問題がないか

### 権限 / アクセス制御

- 権限外操作や想定外アクセスが通らないか

### 外部依存との接続

- API、DB、キュー、外部サービスとの接続が期待どおりか

### 可観測性

- ログ、メトリクス、通知が必要十分か

### 性能 / 処理量耐性

- 想定負荷で許容範囲に収まるか

### 変更容易性

- 将来変更時に壊れやすい箇所を検出できるか
