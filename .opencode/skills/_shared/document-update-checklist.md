# Document Update Checklist

- Confirm the target document type and canonical location.
- Check the related requirement, spec, ADR, validation, or team guide before editing.
- Update the matching `index.md` file.
- Keep wording consistent with the rest of the managed docs.
- Prefer `owox write --text ...` or stdin for small edits; use larger inputs only when needed.
- Run `owox validate` after material document changes.
