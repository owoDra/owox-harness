# 作業モード

## 使い方

- task 系 skill は開始時に、今回の `目的` と `進め方` を確認して task に残す
- `進め方` は `自走` または `対話` の 2 つから選ぶ
- 指定がない場合は `未確定事項` に残し、判断が必要になる前に確認する
- `task-prepare` は後続 task で使う `進め方` まで決めて引き継ぐ

## 自走

- 最初に計画、停止条件、完了条件を共有したうえで、エージェント主導で進める
- 途中では、重要な前提変更、権限外変更、高リスク変更、破壊的変更、仕様変更、機密や外部影響の大きい判断が出た場合だけ確認する
- 通常の進捗共有は続けるが、節目ごとの採否判断は要求しない

## 対話

- 方針、主要な途中結果、次アクション候補を節目ごとに共有し、ユーザーと一緒に進める
- 少なくとも方針確定後、主要結果整理後、次アクション確定前に確認する
- 優先順位、採否、完了判定に関わる判断は省略せず確認する

## 確認に使う情報

- 目的
- 進め方
- 停止条件
- 完了条件
- 今回ユーザー判断が必要になる想定ポイント
