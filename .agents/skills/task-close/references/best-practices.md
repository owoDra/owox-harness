# task-close のベストプラクティス

## 目的

- タスクを再開可能な状態で閉じる
- 成果と残課題を明確に分ける
- 次アクションへ接続する

## 基本原則

- 完了条件で閉じる
- 残課題を隠さない
- 更新済み正本と検証を整理する
- 引き継ぎ情報を短く残す
- ユーザー判断が必要な確認は `request_user_input` ツールで残す

## 推奨フロー

1. `request_user_input` ツールで `goal`、`execution_mode`、完了判定基準を定義する
2. 完了条件の充足を確認する
3. 成果と未完了事項を分ける
4. 次アクションを提案する
5. `tasks/task-*.md` を最終更新する

## execution_mode の使い分け

### agent-led

- 整理観点と停止条件を先に共有してから自律的に進める
- 完了判定が揺れるときだけ `request_user_input` ツールで確認する

### collab-led

- 完了判定、残課題の扱い、次アクションを節目ごとに確認する
- クローズ条件に迷う点は `request_user_input` ツールで明示的に合意を取る

## 良い成果物の条件

- 何が終わったかが短く分かる
- 何が残っているかが明確
- 更新済み正本と検証が追える
- 次 task にそのまま渡せる

## 停止して確認すべきケース

- 完了条件を満たしていない
- 高優先度の残課題がある
- 引き継ぎ先が不明
- 追加の正本更新が必要
