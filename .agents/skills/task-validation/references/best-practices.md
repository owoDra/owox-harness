# task-validation のベストプラクティス

## 基本原則

- まず validation と完了条件を確認する
- 合格条件を先に定義する
- 証跡を残せる検証を優先する
- 未確認を pass と見なさない

## 推奨フロー

1. 検証対象、合格条件、証跡の取り方を定義する
2. validation と関連正本に沿って検証する
3. 項目ごとに結果、証跡、未達、未確認を整理する
4. validation 自体の不足があれば更新要否を残す

## 停止して確認すべきケース

- 合格条件の解釈が割れる
- 重大な未達がある
- 証跡が不足している
- validation 自体の不足が見つかった

## 良い成果物の条件

- 項目ごとに結果が分かる
- 証跡が追える
- 未達と未確認が分かれている
- 次の修正や資料更新に接続できる
