# 共有して確認する検証候補

この候補はプロダクト種別にかかわらず、まず確認対象として提示する。
採用するかどうかと、何で検証するかをユーザーに確認してから `validation.md` に反映する。

## 候補項目

### 仕様整合性

- requirement / spec / architecture と実装内容が矛盾していないか
- 何で検証するかの例: レビュー、静的検証、仕様ベーステスト

### 回帰防止

- 既存機能や既存フローを壊していないか
- 何で検証するかの例: 回帰テスト、スナップショット、既存ケース再実行

### エラー時の安全性

- 失敗時に壊れ方が安全か、異常系の扱いが定義どおりか
- 何で検証するかの例: 異常系テスト、フォルト注入、例外処理テスト

### データ整合性

- 入出力、状態遷移、保存データが壊れないか
- 何で検証するかの例: 状態遷移テスト、永続化テスト、契約テスト

### 境界条件

- 最小値、最大値、空値、重複、順序差などの境界で問題がないか
- 何で検証するかの例: パラメータ化テスト、プロパティテスト

### 権限・アクセス制御

- 権限外操作や想定外アクセスが通らないか
- 何で検証するかの例: 認可テスト、ロール別テスト、統合テスト

### 外部依存との接続

- API、DB、キュー、ファイル、外部サービスとの接続が期待どおりか
- 何で検証するかの例: 契約テスト、統合テスト、スタブ検証

### 可観測性

- ログ、メトリクス、トレース、エラー通知が必要十分か
- 何で検証するかの例: ログ検証、監視テスト、手動確認

### 性能・処理量耐性

- 想定負荷や処理量で許容範囲に収まるか
- 何で検証するかの例: ベンチマーク、負荷試験、プロファイル

### 変更容易性

- 将来変更時に壊れやすい箇所を検出できるか
- 何で検証するかの例: 静的解析、設計レビュー、特性テスト
