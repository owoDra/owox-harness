# 用語集

## Requirement
何を実現するか、なぜ必要か、何が成功かを示す正本。

## Architecture
共通の不変ルール。作業方針。

## Spec
要求を具体的な振る舞い、状態、権限、エラー、フロー、入出力、形式、不変条件、互換条件に落とした資料。

## ADR
あらゆる重要な設計・実装・運用判断の記録。

## Team Guide
チーム別の作法と観点。

## Trace Tag
コードやテストと requirement / spec / ADR を結び付けるタグ。

## Docs
外部共有・納品用に生成される資料。

## Doc Comment
コード内のドキュメンテーションコメント

## ハーネス資料
`.agents/` 以下の `.md` ファイル

## ハーネスライフサイクル
`AGENTS.md` や `.agents/skills` といったハーネスの動作を決めるルールやコンテキスト
