# ルール

- 1 pattern = 1 file
- pattern は複数の spec や実装で再利用したいコード実装、データ構造・設計、UI/UX 設計の共通方針を扱う
- requirement、spec、ADR を置き換えず、それらを横断する一貫性の正本として扱う
- 実プロジェクトで繰り返し使う価値があるものだけを pattern として残す
- pattern のファイル名はカテゴリ別プレフィックス付きで命名する
  - 実装系: `impl-*.md`
  - データ系: `data-*.md`
  - UI 系: `ui-*.md`
  - UX 系: `ux-*.md`
  - API 系: `api-*.md`
  - テスト系: `test-*.md`
  - 運用系: `ops-*.md`
