# 用語集

## ハーネス
`.agents/` 配下にある、エージェントの制約、作業手順、スキル、タスク記録の正本。

## プロジェクト資料
`docs/project/` 配下にある、要件、設計、チームルール、外部連携、検証方針の正本。

## Requirement
何を実現するか、なぜ必要か、何が成功かを示す正本。

## Architecture
プロジェクト全体の不変条件、責務分離、設計方針の正本。

## Spec
要求を具体的な振る舞い、状態、権限、エラー、フロー、入出力、形式、不変条件、互換条件に落とした資料。

## Pattern
複数の spec や実装で再利用するコード実装、データ構造・設計、UI/UX 設計上の共通パターンを整理し、一貫性を保つための正本。

## ADR
あらゆる重要な設計・実装・運用判断の記録。

## Team Guide
チーム別の作法と観点。

## Trace Tag
コードやテストと requirement / spec / ADR を結び付けるタグ。

## Doc Comment
コード内のドキュメンテーションコメント

## Validation
品質を担保するために確認すべき項目、方法、期待結果を定義した正本。

## Tech Stack
採用技術と、その採用判断への参照を整理した正本。
